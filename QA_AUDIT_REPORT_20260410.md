**File:** `scripts/setup-demo-accounts.mjs`

### Issue #2: Unnecessary Regex Escape Character
**File:** `src/contexts/AuthContext.tsx`  
**Line:** 92  
**Error:** ESLint no-useless-escape - Unnecessary escape character `\-`

**Root Cause:**
- Regex pattern had redundant backslash escaping the hyphen character
- In character classes `[...]`, hyphens don't require escaping unless at start/end

**Fix Applied:**
```typescript
// BEFORE
.replace(/[^a-z0-9\-]/g, '')

// AFTER  
.replace(/[^a-z0-9-]/g, '')
```

---

### Issue #3: Unused ESLint Directive
**File:** `src/data/tenantDemoSeed.ts`  
**Line:** 1  
**Warning:** ESLint unused-disable - Directive had no purpose

**Root Cause:**
- `/* eslint-disable */` comment at top with no actual violations to suppress
- This is an auto-generated file that doesn't need blanket disable

**Fix Applied:**

// Auto-generated by scripts/generate-tenant-demo-seed.mjs

### Deployment Method

Used verified **temp-directory workaround** to avoid `.d.ts` file scanning issues:
1. Copy `dist/` to `%TEMP%/fleetpro-pages-dist`
2. Copy required function file (`telegram.ts`)
3. Run wrangler from temp directory
4. Deploy with `--commit-dirty=true` flag

This approach prevents CI/D.ts bundling errors when scanning `functions/node_modules`.
